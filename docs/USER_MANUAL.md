## 1. Message Consolidator란?

### 1.1 한 줄 요약

여러 채널(Slack · WhatsApp · Gmail · Telegram)에 흩어진 업무 메시지를 한 곳으로 모으고, AI가 자동으로 "업무"를 추출·분류해 단일 대시보드에서 관리할 수 있게 해주는 도구입니다.

### 1.2 작동 방식

```
   [Slack · WhatsApp · Gmail · Telegram]
                  │
                  ▼
              Scanner          ← 메시지 수집
                  │
                  ▼
           AI Extraction       ← Gemini 가 업무 추출 / 상태 판별
              (Gemini)
                  │
                  ▼
              Database         ← 사용자별 격리 저장
                  │
                  ▼
              Dashboard        ← 받은/맡긴/참조 업무로 분류
```

AI 가 메시지를 읽고 다음 세 가지 상태로 판별합니다.

| 상태 | 의미 |
|---|---|
| **NEW** | 새로 들어온 업무 — Dashboard 에 신규 항목으로 추가 |
| **UPDATE** | 기존 업무에 대한 답변·진행 상황 — 같은 업무에 누적 |
| **RESOLVE** | 업무가 완료된 것으로 판단됨 — 자동으로 보관함 이동 |

### 1.3 지원 채널과 가능한 작업

| 채널 | 연동 방식 | 사용자 액션 |
|---|---|---|
| Gmail | Google OAuth | 본인이 직접 연동 |
| WhatsApp | QR 코드 페어링 | 본인이 직접 연동 |
| Telegram | 휴대폰 번호 + OTP | 본인이 직접 연동 |
| Slack | 워크스페이스 공용 | **운영자가 사전 설정** — 사용자는 가입만 하면 자동 매핑 |

> **이 인스턴스의 멀티테넌시 동작**
> - 각 사용자의 데이터(메시지·업무·태그)는 이메일 단위로 완전 격리됩니다.
> - WhatsApp · Gmail · Telegram 은 **사용자별로 각자 연동**합니다.
> - Slack 만 워크스페이스 단위 공용입니다 — 운영자가 한 번 설정하면 가입 시 본인 Slack 계정과 자동 매핑됩니다.

## 2. 시작하기

### 2.1 첫 로그인

1. 운영자로부터 받은 인스턴스 URL 에 접속합니다.
2. **Google 로 로그인** 버튼을 클릭합니다.
3. Google 계정 선택 → 권한 동의 화면에서 동의합니다.
4. 로그인이 완료되면 자동으로 대시보드로 이동합니다.

![로그인 화면](./images/login.png)

별도의 회원가입 절차는 없습니다. 첫 Google 로그인 시 사용자 계정이 자동 생성됩니다.

### 2.2 본인 데이터 격리 안내

- 한 인스턴스에 여러 사용자가 가입할 수 있지만, 각자 자신의 **이메일 기준으로 데이터가 격리**됩니다.
- 다른 사용자의 메시지·업무·연동 채널은 보이지 않으며, 영향을 주지도 받지도 않습니다.
- 단, 같은 워크스페이스의 Slack 메시지는 운영자가 설정한 **공용 Bot Token** 을 통해 모든 가입자에게 자동 적용됩니다(메시지 자체는 Slack 의 본인 ID 기준으로 분류).

### 2.3 권장 연동 순서

처음 가입 후 다음 순서로 채널을 연동하시기를 권장합니다.

1. **Gmail** — 가장 안정적이며 즉시 동작
2. **WhatsApp** — QR 1회 스캔으로 완료
3. **Telegram** — `my.telegram.org` 에서 App ID/Hash 발급이 필요해 다소 시간 소요
4. **Slack** — 별도 작업 불필요(자동)

연동된 채널은 대시보드 상단에 "연결됨" 표시가 나타납니다.

## 3. 커넥터별 연동 방법

### 한 곳에서 관리하기 — 설정 → 연결

모든 채널의 연결·재인증·해제는 **설정 → 연결** 화면에서 통합 관리합니다.

진입 경로:
- 헤더의 톱니바퀴(설정) 아이콘 → 좌측 **연결** 탭
- 또는 대시보드 상단의 채널 아이콘(Slack/WhatsApp/Gmail/Telegram) **아무거나 클릭** — 동일하게 설정 → 연결 화면이 열립니다.

대시보드 상단 채널 아이콘의 의미:

| 표시 | 의미 |
|---|---|
| 연결됨 | 정상 동작 중 |
| 연결 안됨 | 연동 필요 |
| 연결 끊김 | 토큰 만료·디바이스 로그아웃 등 — 재연동 필요 |

연결 화면에는 4개 채널 카드가 한 화면에 표시되며, 각 카드에서 다음을 확인·수행할 수 있습니다:
- 연결 상태와 식별 정보(Gmail 이메일, WhatsApp 디바이스명, Telegram 마스킹된 전화번호·App ID, Slack 사용자 ID)
- **연결하기 / 재인증 / QR 재스캔 / App ID·Hash 변경 / 연동 해제 / 로그아웃** 등 채널별 액션

### 3.1 Gmail

**무엇을 하나요?** 본인 Gmail 의 신규 메일을 자동으로 스캔하여 업무 메시지를 추출합니다. 마케팅·뉴스레터는 자동 필터링됩니다.

**사전 준비물**
- 사용 중인 Google 계정(로그인 계정과 달라도 무방)

**연동 절차**
1. **설정 → 연결** 화면을 엽니다(대시보드 상단의 Gmail 아이콘 클릭으로도 진입 가능).
2. **Gmail 카드**에서 **연결하기** 버튼을 클릭합니다.
3. Google OAuth 동의 화면이 열립니다.
4. 요청되는 권한: **Gmail 메일 읽기 전용**(`gmail.readonly`).
5. 동의 후 자동으로 대시보드로 돌아오며 카드가 "연결됨" 으로 표시됩니다.

![Gmail 연동 동의 화면](./images/gmail-consent.png)

**재인증 / 연결 해제**
설정 → 연결 → **Gmail 카드** → **재인증** 또는 **연동 해제** 버튼.

문제가 발생하면 [6.1 Gmail 연결이 끊겼다고 나옴](#61-gmail-연결이-끊겼다고-나옴) 참조.

### 3.2 WhatsApp

**무엇을 하나요?** 본인 WhatsApp 계정의 1:1 채팅·그룹 메시지를 모니터링하여 업무를 추출합니다. WhatsApp Web 과 동일한 디바이스 페어링 방식을 사용합니다.

**사전 준비물**
- 모바일 WhatsApp 앱이 설치된 휴대폰
- 활성화된 WhatsApp 계정

**연동 절차**
1. **설정 → 연결** 화면을 엽니다(대시보드 상단의 WhatsApp 아이콘 클릭으로도 진입 가능).
2. **WhatsApp 카드**에서 **연결하기** 버튼을 클릭합니다 — QR 모달이 열립니다.
3. 휴대폰의 WhatsApp 앱을 엽니다.
4. WhatsApp 앱의 **설정 → 연결된 디바이스 → 디바이스 연결** 메뉴에서 화면의 QR 을 스캔합니다.
5. 스캔이 완료되면 자동으로 카드가 "연결됨" 으로 변경되며, 디바이스명이 표시됩니다.

![WhatsApp QR 화면](./images/whatsapp-qr.png)

**연결 해제 / 재연동**
- 설정 → 연결 → **WhatsApp 카드** → **로그아웃** — 인스턴스 측 세션 해제.
- 설정 → 연결 → **WhatsApp 카드** → **QR 재스캔** — 새 QR 발급 후 재스캔.
- 휴대폰 측에서 끊으려면 WhatsApp 앱의 **연결된 디바이스** 메뉴에서 해당 세션을 제거.

문제가 발생하면 [6.2 WhatsApp QR 만료/로그아웃](#62-whatsapp-qr-만료--로그아웃) 참조.

### 3.3 Telegram

**무엇을 하나요?** 본인 Telegram 계정의 1:1 대화·그룹·채널 메시지를 모니터링합니다. Telegram MTProto 사용자 API 를 사용하므로 본인이 직접 App ID/Hash 를 발급받아야 합니다.

**사전 준비물**
- 활성화된 Telegram 계정과 사용 가능한 휴대폰
- `my.telegram.org/apps` 에서 발급받은 **App ID** 와 **App Hash** (1회만)

#### 3.3.1 App ID / App Hash 발급

1. 브라우저에서 [https://my.telegram.org/apps](https://my.telegram.org/apps) 에 접속합니다.
2. 본인 Telegram 계정의 휴대폰 번호로 로그인합니다(SMS·앱 코드 인증).
3. 신규 애플리케이션 정보를 입력합니다.
   - App title: 자유 입력 (예: "MessageConsolidator")
   - Short name: 자유 입력 (예: "msgc")
   - Platform: Web
4. **Create application** 클릭 후 표시되는 **App ID** (숫자) 와 **App Hash** (32자 문자열) 를 메모합니다.

> **주의**: App ID/Hash 는 본인 외 누구에게도 공유하지 마세요. 인스턴스 외부에 노출되면 본인 Telegram 계정 탈취 위험이 있습니다.

#### 3.3.2 인스턴스에 등록

1. **설정 → 연결** 화면을 엽니다(Telegram 아이콘 클릭으로도 진입 가능).
2. **Telegram 카드**의 **연결하기** 버튼 클릭 — 인증 모달이 열립니다.
3. 발급받은 **App ID** 와 **App Hash** 를 입력하고 **저장하고 계속**.
4. 휴대폰 번호 입력(국가번호 포함, 예: `+821012345678`) → **코드 전송**.
5. Telegram 앱으로 도착한 **인증 코드** 입력 → **확인**.
6. (2단계 인증을 사용 중인 경우) **클라우드 비밀번호** 입력 → **확인**.
7. 카드가 "연결됨" 으로 표시되면 완료. 마스킹된 전화번호와 App ID 가 카드에 노출됩니다.

![Telegram 인증 단계](./images/telegram-auth.png)

**App ID·Hash 변경 / 재인증 / 연결 해제**
설정 → 연결 → **Telegram 카드** 의 다음 버튼:
- **App ID·Hash 변경** — 연결된 상태에서도 인증 정보를 갱신할 수 있습니다.
- **재인증** — 다른 휴대폰 번호로 다시 인증.
- **로그아웃** — 인스턴스 측 세션 해제. App ID/Hash 는 보존됩니다.

문제가 발생하면 [6.3 Telegram 2FA / 코드 재전송](#63-telegram-2fa--코드-재전송) 참조.

### 3.4 Slack

**무엇을 하나요?** 워크스페이스의 채널·DM 메시지를 모니터링하여 본인 Slack 계정 기준으로 업무를 추출합니다.

**사용자 액션**
별도의 연동 절차가 **없습니다**. Slack 은 운영자가 워크스페이스에 설치한 Bot Token 을 통해 동작하며, 본인 Google 로그인 이메일과 같은 이메일이 등록된 Slack 사용자가 자동으로 매핑됩니다.

**확인 방법**
- 대시보드 상단에 "Slack: 모니터링 중" 으로 표시되면 정상입니다.
- 설정 → 연결 → **Slack 카드** 에서 본인에게 매핑된 Slack 사용자 ID 를 확인할 수 있습니다.

**자동 매핑이 되지 않는 경우**
- 본인의 Slack 가입 이메일이 Google 로그인 이메일과 다른 경우 자동 매핑이 실패할 수 있습니다(Slack 카드에 "자동 매핑이 아직 완료되지 않았습니다" 안내 표시).
- 이때는 운영자에게 문의하거나, 설정 → **이름 관리 규칙** 메뉴에서 본인 식별자를 직접 등록할 수 있습니다.

## 4. 기능별 사용 가이드

### 4.1 Dashboard

대시보드 메인 화면은 업무를 세 가지 탭으로 분류합니다.

| 탭 | 의미 |
|---|---|
| **받은 업무** | 본인이 처리해야 할 업무 — 본인이 담당자(Assignee)로 지정된 항목 |
| **맡긴 업무** | 본인이 다른 사람에게 부탁한 업무 — 본인이 요청자(Requester)인 항목 |
| **참조 업무** | 본인이 직접 관련되지 않았지만 참고할 만한 업무 |

각 행은 다음 정보를 표시합니다.

- 채널 아이콘 (Slack/WhatsApp/Gmail/Telegram)
- 룸/스레드 출처
- 업무 내역
- 요청자 / 담당자
- 발생 시간

상단의 **🔄 지금 스캔** 버튼으로 즉시 새 메시지를 가져올 수 있으며, 그 외에는 백그라운드에서 자동 스캔됩니다.

대시보드 상단 **노이즈 필터링** 카드에서 오늘/이번 달 차단된 마케팅 메일 개수를 확인할 수 있습니다.

### 4.2 Task 액션

각 업무 행에서 사용 가능한 액션입니다.

| 액션 | 결과 |
|---|---|
| **✓ 완료** | 업무를 완료 상태로 표시합니다. 보관함의 "완료된 업무" 로 이동. |
| **삭제** (소프트) | 보관함의 "취소한 업무" 로 이동. 복원 가능. |
| **영구 삭제** (하드) | DB 에서 완전 삭제. **되돌릴 수 없습니다.** |
| **편집** | 제목·요청자·담당자를 직접 수정. |
| **병합** | 중복으로 추출된 업무를 하나로 통합. |
| **원문 보기** | 메시지 원본 텍스트를 모달로 표시. |
| **{채널}에서 열기** | 원본 메시지가 있는 Slack/Telegram 등으로 직접 이동. |

#### 보관함

대시보드 → 보관함 메뉴에서 다음을 확인할 수 있습니다.

- **전체 / 완료된 업무 / 취소한 업무 / 병합한 업무**
- 보관함 항목은 **복원** 또는 **영구 삭제** 가능
- **내보내기** 버튼으로 Excel(.xlsx) / CSV / JSON 다운로드

### 4.3 Report (AI 보고서)

기간을 지정하면 AI 가 해당 기간 동안의 업무 활동을 요약한 보고서를 생성합니다.

**생성 절차**
1. 대시보드 → **인사이트** → **보고서** 탭 → **새 보고서 생성**.
2. **시작일 / 종료일** 선택.
3. **생성 및 분석** 클릭.
4. AI 가 비동기로 생성하며, 완료 시 보고서 목록에 추가됩니다(생성 시간이 길어지면 잠시 후 새로고침).

**보고서 내 기능**
- **번역** — 이미 생성된 보고서를 다른 언어로 번역
- **Notion 으로 내보내기** — Notion 페이지로 export (운영자가 Notion 연동을 설정한 경우)
- **삭제** — 개별 보고서 삭제

> 데이터가 많은 기간을 선택하면 토큰 한도 초과로 일부 과거 데이터가 생략될 수 있습니다. 이 경우 보고서 상단에 안내가 표시됩니다.

### 4.4 Insights (통계)

대시보드 → **인사이트** → **통계** 탭에서 다음을 확인할 수 있습니다.

- **누적 완료** 업무 개수
- **최고 집중 시간** — 시간대별 완료 패턴
- **최근 30일 활동** 히트맵
- **시간대별 활동** 라인 차트
- **채널별 분포** 원형 차트
- **업무 완료 추이** (1개월 / 3개월 / 1년 필터)

## 5. 데이터 프라이버시 및 보안

### 5.1 수집 범위

| 채널 | 권한 |
|---|---|
| Gmail | `gmail.readonly` (읽기 전용 — 메일 발송·삭제 불가) |
| WhatsApp | WhatsApp Web 동등 권한 (1:1·그룹 메시지 송수신 가능, 인스턴스는 수신만 사용) |
| Telegram | 사용자 계정 권한 (MTProto 사용자 API — 본인이 가입한 모든 대화에 접근) |
| Slack | 운영자 설정 Bot Token 권한 (일반적으로 채널 메시지·사용자 정보 읽기) |

### 5.2 저장 위치

- 모든 데이터는 **이 인스턴스의 자체 DB(SQLite)** 에 저장됩니다.
- Gemini AI 호출 시 메시지 내용이 Google Gemini API 로 전송되어 추출에 사용됩니다(Google 의 API 정책에 따라 학습 사용 여부가 결정됨).
- 외부의 다른 SaaS 로 데이터가 송출되지 않습니다(Notion 으로 직접 내보내기를 실행한 경우 제외).

### 5.3 연결 해제와 데이터 삭제

| 행위 | 효과 |
|---|---|
| 채널 **연동 해제** | 해당 채널의 신규 메시지 수집 중단. 이미 수집된 업무는 유지. |
| 업무 **삭제**(소프트) | 보관함으로 이동. 복원 가능. |
| 업무 **영구 삭제** | DB 에서 완전 삭제. 되돌릴 수 없음. |
| **로그아웃** | 세션 종료. 데이터·연동 상태는 유지. |
| 계정 자체 삭제 | 운영자에게 별도 요청 — 본 인터페이스에서는 미제공. |

### 5.4 토큰·세션 관리

- 로그인 세션은 HttpOnly 쿠키로 관리되며 자바스크립트로 접근할 수 없습니다.
- Gmail OAuth refresh token, Telegram 세션은 인스턴스 DB 에 암호화 없이 저장됩니다 — 따라서 인스턴스 운영자의 신뢰가 전제됩니다.
- 보안 우려가 있으면 **연동 해제** 후 외부 서비스(Google 계정, my.telegram.org)에서도 해당 세션·앱을 폐기하시기 바랍니다.

## 6. 트러블슈팅

### 6.1 Gmail 연결이 끊겼다고 나옴

**원인**
- Google 계정 비밀번호 변경
- Google 계정 보안 설정에서 앱 액세스 차단
- Refresh token 만료

**해결**
1. 설정 → 연결 → **Gmail 카드** → **재인증** 버튼.
2. 그래도 실패하면 **연동 해제** → **연결하기** 로 처음부터 OAuth 흐름을 다시 진행.
3. 여전히 실패하면 [Google 계정 → 보안 → 타사 앱](https://myaccount.google.com/permissions) 에서 이 인스턴스가 차단돼 있지 않은지 확인.

### 6.2 WhatsApp QR 만료 / 로그아웃

**원인**
- QR 코드는 발급 후 약 1분 내 만료됩니다.
- 휴대폰의 WhatsApp 앱에서 **연결된 디바이스 → 로그아웃** 한 경우.
- 휴대폰이 14일 이상 인터넷에 연결되지 않은 경우 자동 만료.

**해결**
- 설정 → 연결 → **WhatsApp 카드** → **QR 재스캔** 버튼으로 새 QR 발급.
- 휴대폰 측에서 로그아웃된 경우 동일하게 **QR 재스캔** 후 휴대폰의 WhatsApp 앱에서 다시 스캔.

### 6.3 Telegram 2FA / 코드 재전송

**증상**: "2단계 인증이 활성화되어 있습니다. 클라우드 비밀번호를 입력하세요."

이는 본인의 Telegram 계정에 2단계 인증(클라우드 비밀번호)이 켜져 있다는 의미입니다. Telegram 앱 → 설정 → 개인 정보 및 보안 → 2단계 인증에서 설정한 비밀번호를 입력하시면 됩니다.

**코드가 도착하지 않는 경우**
1. Telegram 앱이 켜진 다른 디바이스가 있다면 그쪽 앱으로 코드가 전송됩니다(SMS 가 아님).
2. 휴대폰 번호를 잘못 입력한 경우 → 처음부터 다시 시도.
3. App ID/Hash 가 잘못된 경우 → 설정 → 연결 → **Telegram 카드** → **App ID·Hash 변경** 버튼으로 재입력.

**잘못된 코드를 입력해 잠긴 경우**
연속 실패 시 Telegram 측에서 일시적으로 잠금됩니다. 몇 분 후 설정 → 연결 → **Telegram 카드** → **재인증** 버튼으로 다시 시도하시기 바랍니다.

### 6.4 메시지가 Dashboard 에 안 보임

확인 순서:

1. 채널 상태가 **연결됨** 인가? — 연결 끊김이면 재연동.
2. 메시지가 실제 업무성 내용인가? — AI 가 단순 인사·잡담은 필터링합니다.
3. 노이즈 필터링에 걸렸는가? — Gmail 의 마케팅·뉴스레터는 자동 차단됩니다(노이즈 필터링 카드에서 차단 개수 확인).
4. **🔄 지금 스캔** 버튼을 눌러 강제 스캔.
5. 그래도 누락이 있으면 운영자에게 문의.

### 6.5 AI 추출 결과가 비어있음 / 잘못 분류됨

- AI 모델은 100% 정확하지 않습니다. 실제 담당자·내용이 다르게 추출됐다면 행의 **편집** 버튼으로 직접 수정하실 수 있습니다.
- 본인이 "받은 업무" 가 아닌데 받은 업무 탭에 들어와 있는 경우 → 설정 → **내 별칭 설정** 에서 본인이 자주 불리는 이름·별명을 추가하시면 정확도가 올라갑니다.
- 같은 업무가 여러 건으로 중복 추출됐다면 **병합** 기능을 사용하세요.

## 7. FAQ

**Q. 다른 사용자의 메시지·업무가 제 대시보드에 보일 수 있나요?**
A. 보이지 않습니다. 데이터는 본인 이메일 단위로 격리됩니다. 단, Slack 의 경우 본인이 참여한 채널·DM 의 메시지만 본인 기준으로 필터링되어 표시됩니다.

**Q. 한 Gmail 계정을 여러 사람이 공유해도 되나요?**
A. 권장하지 않습니다. 본 인스턴스는 "1 사용자 = 1 Google 로그인 이메일" 을 가정합니다. 공유 메일함을 다루려면 운영자에게 별도 워크플로우를 문의하세요.

**Q. Gemini 가 제 메시지 내용을 학습하나요?**
A. 메시지 내용은 업무 추출을 위해 Gemini API 로 전송됩니다. 학습 사용 여부는 Google 의 API 데이터 정책에 따릅니다. 민감한 메시지는 사전에 채널 단위로 연동을 차단하실 수 있습니다.

**Q. 채널 연동을 끊으면 기존 업무도 사라지나요?**
A. 사라지지 않습니다. 연동 해제는 **신규 메시지 수집만 중단**합니다. 기존 업무를 모두 지우려면 보관함에서 영구 삭제하거나 운영자에게 계정 삭제를 요청하세요.

**Q. Slack 메시지가 어떻게 자동으로 들어와있나요?**
A. Slack 만 워크스페이스 단위 공용 Bot Token 으로 동작합니다. 운영자가 워크스페이스에 Bot 을 설치하면, 본인이 가입할 때 본인 Slack 이메일과 자동 매핑됩니다.

**Q. WhatsApp 그룹 메시지도 다 들어오나요?**
A. 네. 본인이 참여 중인 모든 1:1·그룹 채팅이 모니터링 대상입니다. 특정 대화를 제외하려면 현재는 별도 차단 기능이 없으므로 본인 WhatsApp 앱에서 해당 채팅에서 나가는 방법을 사용해야 합니다.

**Q. 보관함에서 복원할 수 있나요?**
A. 소프트 삭제·완료 처리된 항목은 보관함에서 **복원** 으로 되돌릴 수 있습니다. 단, **영구 삭제** 한 항목은 복원 불가합니다.

**Q. Report 를 다른 언어로 받고 싶어요.**
A. 보고서 상세에서 **번역** 버튼을 사용하시면 됩니다. 원본 보고서는 그대로 유지되고 번역본이 추가됩니다.

**Q. 토큰 사용량은 어디서 확인하나요?**
A. 대시보드 → 설정 → **토큰 사용량 모니터링** 메뉴에서 오늘·이번 달 Gemini API 토큰 사용량과 추정 비용을 확인할 수 있습니다.

**Q. 이름이 잘못 표시돼요(예: 영문 이름이 한글로 바뀌어야 함).**
A. 설정 → **표기명 변경(Display Name Override)** 에서 원본 이름과 표시할 이름을 매핑하면 대시보드 전반에 적용됩니다.
